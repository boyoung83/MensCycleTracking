# 🌙 달의 기록 · 생리주기 트래커

광고 없는 개인용 생리주기 관리 웹앱(PWA). 서버 없이 동작하고, 데이터는 내 폰에만 저장됩니다.

## 기능
- **달력 기록**: 날짜를 탭해 생리 시작일·종료일 입력, "오늘 시작" 빠른 버튼
- **현황 대시보드**: 평균 주기, 평균 생리 기간, 마지막 시작일, **다음 예정일**, 주기 변동 그래프, 가임기 추정
- **알림**: 예정일 **2일 전** + **예정일 당일 "시작했나요?" 확인** 알림
  - 앱을 안 켜도 뜨도록 **구글 캘린더**에 자동 등록 (선택)
  - 앱 사용 중에는 로컬 알림도 동작
- **데이터 백업**: JSON 파일로 내보내기/불러오기 (기기 변경·초기화 대비)

## 설치 방법 (안드로이드)
1. GitHub 저장소 **Settings → Pages → Source**를 **GitHub Actions**로 설정
2. 잠시 뒤 배포되는 주소(예: `https://<계정>.github.io/<저장소이름>/`)를 크롬으로 접속
3. 크롬 메뉴 → **홈 화면에 추가** → 앱 아이콘 생성
4. 아이콘을 열어 앱처럼 사용, 첫 실행 시 알림 권한 허용

> Actions 탭의 "Deploy PWA to GitHub Pages" 실행 로그 맨 아래 `page_url`에서 정확한 주소를 확인하세요.

## 백그라운드 알림용 구글 캘린더 연동 (선택, 1회 설정)
웹앱은 완전히 닫혀 있을 때 스스로 알림을 띄우는 것이 브라우저마다 제한적입니다.
그래서 예정일 알림을 **내 구글 캘린더 일정**으로 등록해, 캘린더 앱이 확실히 알려주도록 합니다.

1. [Google Cloud Console](https://console.cloud.google.com) 접속 → 새 프로젝트 생성
2. **API 및 서비스 → 라이브러리** → **Google Calendar API** 사용 설정
3. **OAuth 동의 화면**: 사용자 유형 "외부", 본인 이메일을 **테스트 사용자**로 추가
4. **사용자 인증 정보 → OAuth 클라이언트 ID → 웹 애플리케이션**
   - **승인된 자바스크립트 원본**에 이 앱의 주소 origin을 추가
     (앱 설정 화면에 정확한 origin이 표시됩니다)
5. 생성된 **클라이언트 ID**를 앱 **설정 → 구글 OAuth 클라이언트 ID** 칸에 붙여넣기
6. **"구글 캘린더 연동 & 동기화"** 버튼 클릭 → 로그인/동의

이후 기록을 추가할 때마다 예정일이 다시 계산되어 캘린더 일정이 자동 갱신됩니다.

## 데이터/개인정보
- 모든 기록은 브라우저 `localStorage`에만 저장됩니다(서버 전송 없음).
- 구글 캘린더 연동 시에만, 예정일 일정이 본인 캘린더에 생성됩니다.
- 기기 초기화·브라우저 데이터 삭제 시 기록이 사라지니 **정기적으로 백업**하세요.

## 기술
순수 정적 웹앱 — HTML/CSS/JS, 서비스워커, Web App Manifest. 빌드 도구·의존성 없음.
